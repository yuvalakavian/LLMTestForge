Kotlin Basics

Kotlin היא שפת תכנות מודרנית מבוססת JVM, פשוטה, בטוחה ועם תמיכה בפרדיגמות תכנות פונקציונליות. להלן סקירה מקיפה עם דגש על יסודות השפה, כולל הגדרות משתנים, עקרונות בטיחות טיפוס, עבודה עם ערכי null, פונקציות ותחביר, טיפוסי נתונים, מבני בקרה, ומבני נתונים.

הוראות התקנה

  • הורד והתקן את IntelliJ IDEA מהאתר של JetBrains.
  • השתמש ב-Android Studio Kotlin REPL למצבי פיתוח וניסוי.
  • במקרה של בעיות בפתיחת REPL, יש לעבור ל-Run > Edit Configurations... > Templates > Java Scratch ולהגדיר את Shorten command line to @argfile (Java 9+).

הגדרות משתנים

var name: String

שימוש ב-vals

ב-Kotlin, 'val' מציין ערכים קבועים שלא ניתן לשנותם לאחר הקצאה. תמיד מעדיפים להשתמש ב'val' אלא אם חייבים להשתמש ב'var', מכיוון שזה מבטיח בטיחות ניסוחית ותחזוקה קלה יותר של הקוד.

עבודה עם null

ב-Kotlin, לא ניתן להקצות null למשתנים רגילים כדי למנוע NullPointerException. אם יש צורך במשתנה היכול להכיל null, יש להגדירו כ-String? או Int?. כדי לגשת למשתנים כאלה, ניתן להשתמש במפעילים ?. ו-!! אשר מאפשרים גישה בטוחה לערכים.

מבני בקרה

ל-Kotlin יש מבני בקרה כמו if, when, ו-loops המאפשרים כתיבת קוד קריאה ותחזוק. מבנה ה-if מחזיר ערכים ובכך חוסך את הצורך במפעיל תנאי מסורתי. ה-when דומה ל-switch אך ללא צורך במילות מפתח כמו case ו-break.

פונקציות וניהול חריגות

  • 'fun' היא מילת המפתח להכרזת פונקציות ב-Kotlin. הפונקציות יכולות להחזיר ערכים, לערך פרמטרים ברירת מחדל, ולתמוך ב-varargs.
  • Kotlin תומכת בחריגות לא בדוקות, ולכן אין צורך להשתמש במילת המפתח throws.

מחלקות ובנאים

  • הקדמה: ב-Kotlin, ניתן להגדיר מחלקות עם בנאי ראשוני ו/או משני. הבנאי הראשוני מופיע בכותרת המחלקה ולא מכיל קוד. ניתן להוסיף פרקי אתחול באמצעות המילה init.
  • מילת מפתח open: כברירת מחדל, מחלקות ב-Kotlin מסומנות כ-final ולא ניתן להורישן. יש להשתמש במילת המפתח 'open' כדי לאפשר הורשה.
  • מחלקות מופשטות: Kotlin מאפשרת יצירת מחלקות מופשטות שכוללות שיטות מופשטות ולא מופשטות. מחלקות אלו משמשות כבסיס למחלקות נגזרות.

תפיסת אובייקטים ומחלקות

  • Kotlin תומכת בתחום מחלקות ואובייקטים עם תכונות כמו מחלקות סינגלטון, שימוש באובייקטים אנונימיים, ומחלקות מלוות (Companion Objects).
  • אובייקטים אנונימיים שימושיים למקרים חד-פעמיים והם מוגדרים באמצעות ביטוי ולא שם.

שימוש ב-Extension Functions

Kotlin מאפשרת להרחיב מחלקות באמצעות פונקציות הרחבה (extension functions), המאפשרות להוסיף פונקציות למחלקות קיימות ללא ירושה.
